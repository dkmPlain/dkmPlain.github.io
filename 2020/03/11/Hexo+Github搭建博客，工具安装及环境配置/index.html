Hexo+Github搭建博客，工具安装及环境配置

一、工具准备

  • Node.js 环境和npm (Node Package Manager,node包管理器)
  • Git 环境
  • Hexo

二、开始配置

  1. 安装Node.js (版本：12.16.1)，下载：官网地址，国内：镜像地址，注意安装Node.js会包含环境变量及npm的安装，安装后，检测Node.js是否安装成功，在命令行中输入 node -v 来显示软件版本。

  2. Git (版本：2.25)， 下载：官网地址，国内：镜像地址， 这里编辑器选择Notepad++，其它选项默认即可。

  3. 安装Hexo，Win+R打开 Windows Command Line (cmd 命令)，输入：npm install -g hexo-cli，安装完成后，Hexo -v 显示版本。可以参考官网：hexo.io>zh-cn>docs

    关于hexo-cli：command line interface的缩写。按Hexo官网的安装指南，只安装命令行方式即可。

  4. 在合适的目录下创建网站文件夹，例如：web-host。Win+R -cmd, 切换到web-host目录（空目录，不要有其它文件）。然后输入： hexo init， 初始化站点文件夹。

  5. 输入：hexo g，生成静态网页，在站点根目录下生成public文件夹，编译.md文件到.html文件。

  6. 输入：hexo s，启动server。

  7. 本地浏览器输入：localhost:4000，查看页面。

三、内容推送到Github

  1. 修改“_config.yml”文件。站点文件夹web-host下，文件末尾，修改文件内容：

    deploy:
    type: git
    repository: https://github.com/dkmPlain/dkmPlain.github.io
    branch: master

  2. 安装deploy-git，部署命令。npm install hexo-deployer-git –save

  3. 配置Git自己的用户名和Email地址。

    git config –global user.name “xxx”
    git config –global user.email “xxx@gmail.com“

  4. 也可以直接修改 web-host>deploy_git>.git 目录下的config文件（文件浏览器需要去掉隐藏选项），添加以下内容。

    [user]

    email = xxx@gmail.com
name = xxx

  5. 然后输入命令：

    hexo clean (#清除缓存)

    hexo g

    hexo d (hexo deploy #部署)

  6. 输入用户名，密码，部署完成后，可以打开浏览器，输入github的仓库路径，即 http://xxxx.github.io即可访问。

四、配置使用ssh key免密部署

  1. 确认已经为Git配置了用户名和邮箱地址。如：

    git config –global user.name “xxx”
    git config –global user.email “xxx@gmail.com“

  2. 修改Hexo文件夹的_config.yml配置文件，将Git克隆的方式由HTTP协议URL改为SSH协议URL，如下：

    repository: git@github.com:username/username.github.io.git

  3. 确认是否存在ssh密钥：

    Horse@DESKTOP-7OP73RU MINGW64 ~
    $ cd ~/.ssh
    bash: cd: /c/Users/Horse/.ssh: No such file or directory

  4. 使用ssh-keygen命令生成github密钥：

    ssh-keygen -t rsa -C “xxx@gmail.com“

    按提示，默认选项，回车即可。

    Horse@DESKTOP-7OP73RU MINGW64 ~
    $ ssh-keygen -t rsa -C “xxx@gmail.com“
    Generating public/private rsa key pair.
    Enter file in which to save the key (/c/Users/Horse/.ssh/id_rsa):
    Created directory ‘/c/Users/Horse/.ssh’.
    Enter passphrase (empty for no passphrase):
    Enter same passphrase again:
    Your identification has been saved in /c/Users/Horse/.ssh/id_rsa
    Your public key has been saved in /c/Users/Horse/.ssh/id_rsa.pub
    The key fingerprint is:
    SHA256:EFSq1tq78X8cUk1hn86LWBztQvCF6Tj26kr69Y/44A4 xxx@gmail.com
    The key’s randomart image is:
    +—[RSA 3072]—-+
    | .+o=o++. |
    | . = +++ . |

    | . o . o.o . |
    | . E =.. . |
    | o *.+ . |
    | ..===.+.. |
    +—-[SHA256]—–+

  5. 使用git命令复制id_rsa.pub的文件内容：

    $ clip < ~/.ssh/id_rsa.pub

  6. 在github中设置ssh key，打开github账号，右上角Settings ->左边SSH and GPG keys，选择New SSH key，右键粘贴 (注意确认代码前后不要留有空格)，点击Add SSH key。

  7. 输入 ssh -T git@github.com 测试，显示提示：

    The authenticity of host ‘github.com (13.229.188.59)’ can’t be established.
    RSA key fingerprint is SHA256:nThbg6kXUpJWGl7E1IGOCspPPomTxdCARLviKw6E5S.
    Are you sure you want to continue connecting (yes/no/[fingerprint])? yes

  8. 输入yes回车即可。
