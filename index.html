Hexo+Github搭建博客，工具安装及环境配置

一、工具准备

  • Node.js 环境和npm (Node Package Manager,node包管理器)
  • Git 环境
  • Hexo

二、开始配置

  1. 安装Node.js (版本：12.16.1)，下载：官网地址，国内：镜像地址，注意安装Node.js会包含环境变量及npm的安装，安装后，检测Node.js是否安装成功，在命令行中输入 node -v 来显示软件版本。

  2. Git (版本：2.25)， 下载：官网地址，国内：镜像地址， 这里编辑器选择Notepad++，其它选项默认即可。

  3. 安装Hexo，Win+R打开 Windows Command Line (cmd 命令)，输入：npm install -g hexo-cli，安装完成后，Hexo -v 显示版本。可以参考官网：hexo.io>zh-cn>docs

    关于hexo-cli：command line interface的缩写。按Hexo官网的安装指南，只安装命令行方式即可。

  4. 在合适的目录下创建网站文件夹，例如：web-host。Win+R -cmd, 切换到web-host目录（空目录，不要有其它文件）。然后输入： hexo init， 初始化站点文件夹。

  5. 输入：hexo g，生成静态网页，在站点根目录下生成public文件夹，编译.md文件到.html文件。

  6. 输入：hexo s，启动server。

  7. 本地浏览器输入：localhost:4000，查看页面。

三、内容推送到Github

  1. 注册Github账户，创建一个Repository。Repository Name必须是URL 根目录，默认值：

    username.github.io

  2. 修改“_config.yml”文件。站点文件夹web-host下，文件末尾，修改文件内容：

    deploy:
    type: git
    repository: https://github.com/dkmPlain/dkmPlain.github.io
    branch: master

  3. 安装deploy-git，部署命令。npm install hexo-deployer-git –save

  4. 配置Git自己的用户名和Email地址。

    git config –global user.name “xxx”
    git config –global user.email “xxx@gmail.com“

  5. 也可以直接修改 web-host>deploy_git>.git 目录下的config文件（文件浏览器需要去掉隐藏选项），添加以下内容。

    [user]

    email = xxx@gmail.com
name = xxx

  6. 然后输入命令：

    hexo clean (#清除缓存)

    hexo g

    hexo d (hexo deploy #部署)

  7. 输入用户名，密码，部署完成后，可以打开浏览器，输入github的仓库路径，即 http://xxxx.github.io。

四、配置使用ssh key免密部署

  1. 确认已经为Git配置了用户名和邮箱地址。如：

    git config –global user.name “xxx”
    git config –global user.email “xxx@gmail.com“

  2. 修改Hexo文件夹的_config.yml配置文件，将Git克隆的方式由HTTP协议URL改为SSH协议URL，如下：

    repository: git@github.com:username/username.github.io.git

  3. 开始菜单->git->git bash，使用git bash命令行，确认是否存在ssh密钥：

    Horse@DESKTOP-7OP73RU MINGW64 ~
    $ cd ~/.ssh
    bash: cd: /c/Users/Horse/.ssh: No such file or directory

  4. 使用ssh-keygen命令生成github密钥：

    ssh-keygen -t rsa -C “xxx@gmail.com“

    按提示，默认选项，回车即可。

    $ ssh-keygen -t rsa -C “xxx@gmail.com“

  5. 使用git命令复制id_rsa.pub的文件内容：

  6. 在github中设置ssh key，打开github账号，右上角Settings ->左边SSH and GPG keys，选择New SSH key，右键粘贴 (注意确认代码前后不要留有空格)，点击Add SSH key。

使用坚果云Git仓库同步Hexo网站数据

使用坚果云做文件同步的时候，如果简单的将Hexo网站文件夹做坚果云同步文件，坚果云会提示不推荐使用坚果云存储代码。坚果云官方微博给出了如下解释：

不少程序员用户，直接将自己的代码放在坚果云同步文件夹中，每次修改自动同步，这样做有几个问题：

  • 坚果云的版本历史是自动保存的，不能添加版本信息。对于代码而言，不方便找回和比对历史代码；
  • 专业的代码管理工具（Git/Hg）能够很容易查看用户对所有代码进行的每次修改，易于分析bug的来源。而坚果云不能提供这样的功能。
  • 如果将eclipse, Visual Studio等IDE的workspace直接放于坚果云中，假如同步不及时造成冲突，而IDE自动生成文件的冲突无法解决，可能导致IDE无法正常打开。
  • 编译，重构等自动化工作引发的同步，容易导致大量的带宽占用。

对于这种问题，坚果云给出了两种方案，一是使用目前国内外成熟的代码托管服务，如下：

国内：coding.net/csdn/开源中国

国外：bitbucket (免费支持私有)/gitlab/github

第二种方案就是在坚果云的同步文件夹中部署git仓库，使用git同步。具体方法：

  1. 坚果云同步文件夹下，例如：Nutstore，创建文件夹。

    git\web-host.git (可以是任意xxx.git名称)

  2. 初始化一个裸仓库，即：不含有工作区的仓库。

    ~\Nutstore\git\web-host.git>git init –bare

  3. 在Hexo网站根目录下创建git仓库。

    ~\web-host>git init (初始化参考，在目录下创建.git文件夹)

    ~\web-host>git add . (#添加所有修改到暂存区，包括添加文件/修改文件/删除文件)

    ~\web-host>git commit -m “first” (#提交暂存区到仓库区)

  4. 为Nutstore中的仓库添加别名。

    ~\web-host>git remote add nutstore ..\Nutstore\git\web-host.git

  5. 将网站数据Push到Nutstore仓库中的master分支。

    ~\web-host>git push nutstore master

  6. 可以在Nutstore的仓库中定期清理，使用gc命令，减少文件数目节约空间。

    例如：~/Nutstore/git/project.git> git gc

附：git教程 https://www.bookstack.cn/books/git-tutorial

坚果云博客：http://blog.jianguoyun.com/?p=2729 坚果云+代码管理工具，有效管理您的代码（程序员必读）
